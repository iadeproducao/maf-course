# 06 - Uso do uv

O `uv` é o gerenciador de pacotes recomendado para este projeto. Ele usa o arquivo `pyproject.toml` para declarar dependências e fornece comandos simples para sincronizar ambientes e executar scripts.

## Por que usar uv?

- Gerencia dependências em `pyproject.toml`.
- Cria e mantém um ambiente virtual consistente.
- Oferece comandos diretos para executar scripts com dependências isoladas.
- Suporta lockfile e exportação, facilitando o controle de versões.

## Passos básicos

1. Instalar o `uv`:

```bash
python -m pip install uv
```

2. Sincronizar as dependências do projeto:

```bash
uv sync
```

3. Executar um exemplo:

```bash
uv run python examples/01-primeiro-agente.py
```

## Dependências do projeto

O `uv` lê as dependências do `pyproject.toml`. Para este curso, as dependências principais são:

- `openai`
- `python-dotenv`
- `requests`

## Dicas úteis

- Para adicionar uma nova dependência:

```bash
uv add <pacote>
```

- Para remover uma dependência:

```bash
uv remove <pacote>
```

- Para gerar ou atualizar o lockfile:

```bash
uv lock
```

- Para executar um comando dentro do ambiente do projeto:

```bash
uv run <comando>
```

## Observações

O `uv` também permite usar scripts Python diretamente com dependências incorporadas em um arquivo script (`uv run script.py`). Mas, neste projeto, a abordagem adotada é manter a configuração centralizada em `pyproject.toml`.
